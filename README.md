# InfraCoreCRM — Full-Stack Платформа Управления Инфраструктурой ЦОД

Современная CRM-система, разработанная для автоматизации учета серверного оборудования, обработки клиентских заявок на аренду/покупку мощностей дата-центра и администрирования пользователей с разграничением прав доступа.

## 🛠 Технологический стек

### Frontend
- **Ядро:** React 19, React Router DOM 7
- **Управление состоянием:** Redux Toolkit (Thunk)
- **Работа с формами и валидация:** React Hook Form, Yup
- **Стилизация:** CSS Modules (адаптивная верстка)
- **Сборщик:** Vite

### Backend
- **Платформа:** Node.js, Express
- **База данных:** MongoDB Atlas (ORM Mongoose)
- **Безопасность:** JWT (JSON Web Tokens), шифрование паролей через bcrypt
- **Загрузка файлов:** Multer


💻 Локальное развертывание и запуск
### 1. Клонирование репозитория
Откройте терминал и выполните команду:

```bash
git clone [https://github.com/grifon2517/InfraCoreCRM-fullStack.git](https://github.com/grifon2517/InfraCoreCRM-fullStack.git)
cd InfraCoreCRM-fullStack
```
2. Настройка и запуск Сервера (Backend)

    Перейдите в директорию бэкенда:

Bash

   cd back

    Установите зависимости:

Bash

   npm install

    Создайте локальный файл конфигурации .env на основе шаблона .env.example:

Фрагмент кода

   PORT=5000
   MONGO_URI=<your_mongodb_connection_string>
   JWT_SECRET=<your_jwt_secret>

    Наполнение базы данных (Сидирование):
    Для автоматического создания тестовых учетных записей (Администратора и Пользователя) запустите готовый скрипт:

Bash

   node config/seed-users.js

    Запустите сервер в режиме разработки:

Bash

   npm run dev

3. Настройка и запуск Интерфейса (Frontend)

    Откройте новое окно терминала и перейдите в директорию фронтенда:

Bash

   cd ../front

    Установите зависимости:

Bash

   npm install

    Создайте локальный файл конфигурации .env на основе шаблона .env.example:

Фрагмент кода

   VITE_API_URL=http://localhost:5000/api
   VITE_SERVER_URL=http://localhost:5000

    Запустите клиентский сервер разработки Vite:

Bash

   npm run dev


🔑 Учетные записи для тестирования функционала

После успешного выполнения скрипта сидирования базы данных (seed-users.js), в систему можно войти под двумя предустановленными ролями:
1. Роль: Администратор (Full CRUD Access)

Позволяет управлять каталогом оборудования, редактировать конфигурации, изменять статусы заявок и просматривать логи.

    Логин: admin

    Пароль: 123123

2. Роль: Пользователь (Client Access)

Позволяет просматривать доступную инфраструктуру, формировать заявки на аренду/сервис и отслеживать их статус в личном кабинете.

    Логин: user

    Пароль: 123123
``
    ## 🛡 Особенности архитектуры и безопасности
- **Role-Based Access Control (RBAC):** Доступ к административной панели защищен как на клиенте (клиентский роутинг React Router), так и на сервере с помощью кастомных middleware авторизации. Запросы к API от пользователей без роли `admin` жестко блокируются сервером.
- **Безопасность данных:** Пароли пользователей никогда не хранятся в открытом виде — при регистрации и сидировании они проходят обсчет хэш-функцией `bcrypt` с солью.
- **Архитектурная чистота:** Использование монорепозитория позволило разграничить зоны ответственности фронтенда и бэкенда, сохранив при этом удобство ведения общей истории разработки в Git.
